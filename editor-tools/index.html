Editor tools

Human-in-the-loop helpers for curating public #ieeedataport posts. These tools do not scrape LinkedIn automatically — you browse and capture with consent.

1. LinkedIn capture bookmarklet

While logged into LinkedIn, open a post from #ieeedataport search , then click the bookmarklet. A linkedin-staging.csv row is copied to your clipboard.

IEEE DP Capture

Drag this button to your bookmarks bar (Chrome: show bookmarks bar → drag link).

  1. Install the bookmarklet (drag link above).
  2. Open a LinkedIn post that mentions IEEE DataPort or #ieeedataport.
  3. Click IEEE DP Capture — CSV row copies to clipboard.
  4. Paste into data/linkedin-staging.csv in the repo.
  5. Commit and push, or wait for the daily pipeline after merge.

2. Member form import

Members can submit via the Submit page, a GitHub issue, or a Google Form. Download the form CSV and import: 

python3 scripts/import_form_export.py data/inbox/member-form-export.csv
python3 scripts/merge_staging.py

See docs/MEMBER_SUBMISSION.md for Google Form field names and setup.

3. Automated imports

  • python3 scripts/import_ieee_stories.py — all case studies in data/ieee-story-sources.json
  • python3 scripts/twitter_import.py — @IEEEDataPort (needs TWITTER_BEARER_TOKEN)
  • python3 scripts/facebook_import.py — IEEE DataPort Page (needs FACEBOOK_PAGE_ACCESS_TOKEN)
  • python3 scripts/linkedin_import.py — IEEE org page (needs LinkedIn API)

4. Pipeline commands

  • python3 scripts/merge_staging.py — CSV → entries.json
  • python3 scripts/score_entries.py --apply — content filter
  • python3 scripts/enrich.py — sentiment/topics
  • ./scripts/daily-pipeline.sh — full refresh + deploy trigger
